模型微调 vs 模型蒸馏：区别、主流方法与多模态微调实践

最后更新于2026年6月24日 晚上

模型微调和模型蒸馏是两种常被混淆的技术，但目标完全不同。本文厘清两者区别，介绍主流微调方法，并重点讲解多模态模型的微调策略。

1. 微调 vs 蒸馏：根本区别

核心对比

维度 模型微调（Fine-tuning） 模型蒸馏（Distillation）
目标 让模型适应新任务/领域 将大模型的知识压缩到小模型
输入 任务标注数据 教师模型的输出（软标签）
输出 同架构、适配新任务的模型 更小、更快的学生模型
模型大小 不变 变小
典型场景 GPT → 对话助手 BERT-large → BERT-small

微调的本质

微调是在预训练权重基础上，用少量任务数据继续训练，调整模型参数使其适应目标任务：

θ∗=arg⁡min⁡θLtask(fθ,Dfinetune)\theta^* = \arg\min_\theta \mathcal{L}_{\text{task}}(f_\theta, \mathcal{D}_{\text{finetune}}) θ∗=argθmin​Ltask​(fθ​,Dfinetune​)

初始化为预训练权重 θ0\theta_0θ0​，而非随机初始化。

蒸馏的本质

蒸馏让小模型（学生）模仿大模型（教师）的输出分布：

LKD=α⋅LCE(y,ps)+(1−α)⋅T2⋅KL(ptT∥psT)\mathcal{L}_{\text{KD}} = \alpha \cdot \mathcal{L}_{\text{CE}}(y, p_s) + (1-\alpha) \cdot T^2 \cdot \text{KL}(p_t^T \| p_s^T) LKD​=α⋅LCE​(y,ps​)+(1−α)⋅T2⋅KL(ptT​∥psT​)

其中 TTT 是温度系数，ptTp_t^TptT​ 是教师的软标签，psTp_s^TpsT​ 是学生的软标签。

2. 主流模型微调方法

2.1 全量微调（Full Fine-tuning）

更新模型所有参数，效果最好但代价最高。

缺点：175B 的模型全量微调需要数百 GB 显存，且容易灾难性遗忘。

2.2 LoRA（Low-Rank Adaptation）

LoRA 是目前最流行的参数高效微调方法（Hu et al., 2021）。

核心思想：冻结原始权重，在每个线性层旁边添加低秩分解矩阵：

W′=W0+ΔW=W0+BAW' = W_0 + \Delta W = W_0 + BA W′=W0​+ΔW=W0​+BA

其中 B∈Rd×rB \in \mathbb{R}^{d \times r}B∈Rd×r，A∈Rr×kA \in \mathbb{R}^{r \times k}A∈Rr×k，秩 r≪min⁡(d,k)r \ll \min(d, k)r≪min(d,k)。

参数量对比（以 LLaMA-7B 为例）：

方法 可训练参数 显存需求
全量微调 7B ~112 GB
LoRA (r=8) ~4M ~16 GB
LoRA (r=64) ~32M ~20 GB

2.3 QLoRA

QLoRA（Dettmers et al., 2023）在 LoRA 基础上加入 4-bit 量化，让消费级 GPU 也能微调 65B 模型。

关键技术：

  • NF4 量化：专为正态分布权重设计的 4-bit 格式
  • 双重量化：对量化常数再次量化，进一步节省内存
  • 分页优化器：用 CPU 内存处理 GPU 内存峰值

2.4 Prefix Tuning / Prompt Tuning

在输入序列前添加可学习的"软提示"向量，模型参数完全冻结：

输入=[p1,p2,...,pk⏟可学习前缀,x1,x2,...,xn]\text{输入} = [\underbrace{p_1, p_2, ..., p_k}_{\text{可学习前缀}}, x_1, x_2, ..., x_n] 输入=[可学习前缀p1​,p2​,...,pk​​​,x1​,x2​,...,xn​]

适用场景：模型完全不能修改（如 API 调用），只能通过 prompt 适配。

2.5 方法对比总结

方法 可训练参数比例 效果 推理开销
全量微调 100% 最好 无
LoRA 0.1%-1% 接近全量 可合并，无额外开销
QLoRA 0.1%-1% 略低于 LoRA 量化推理更快
Prefix Tuning <0.1% 中等 额外前缀计算
Prompt Tuning <0.01% 较弱 无

3. 多模态模型的微调方法

多模态模型（如 LLaVA、BLIP-2、InstructBLIP）的微调有其特殊性：需要同时处理视觉编码器、桥接模块和语言模型三个部分。

3.1 三种微调策略

策略一（最常用）：视觉编码器已经很强，冻结节省计算，只训练桥接模块和 LLM。

策略二：适合视觉任务为主，语言能力要求不高的场景。

策略三：效果最好，LLM 部分通常用 LoRA 降低成本。

3.2 LLaVA 的微调流程

LLaVA（Liu et al., 2023）是最具代表性的多模态微调方案，分两阶段：

阶段一：特征对齐预训练

只训练线性投影层，用 595K 图文对数据，让视觉特征对齐到语言空间。

阶段二：端到端指令微调

用 158K 视觉指令数据，训练投影层 + LLM（LoRA），让模型学会遵循视觉指令。

总结

  • 微调：适配任务，模型大小不变；蒸馏：压缩知识，模型变小
  • LoRA 是当前最实用的微调方法，0.1% 参数量达到接近全量微调的效果
  • 多模态微调的关键是决定冻结哪些部分：视觉编码器通常冻结，桥接模块必须训练，LLM 用 LoRA 平衡效果与成本
