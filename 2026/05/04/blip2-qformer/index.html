BLIP-2：用 Q-Former 桥接视觉与语言大模型

最后更新于2026年5月5日 下午

BLIP-2 是 Salesforce 于 2023 年提出的多模态框架，核心创新是用轻量级的 Q-Former 模块，以极低的训练成本将冻结的视觉编码器与冻结的语言大模型连接起来。

1. 核心思想：冻结大模型，只训练桥接模块

训练多模态大模型面临的困境：

  • 视觉编码器（如 CLIP ViT）已经很强，重新训练代价极高
  • 语言大模型（如 OPT、FlanT5）已经很强，微调会破坏语言能力
  • 两者的特征空间完全不同，如何对齐？

BLIP-2 的解法：只训练一个轻量的 Q-Former 作为桥梁，两端的大模型全部冻结。

2. Q-Former 架构详解

Q-Former（Querying Transformer）是 BLIP-2 的核心创新。

2.1 结构

关键设计： 32 个可学习的查询向量（Query）通过交叉注意力从图像特征中”提问”并提取信息，将 257 个图像 token 压缩为 32 个，大幅降低 LLM 的输入长度。

2.2 两阶段训练

第一阶段：视觉-语言表示学习（配合冻结的 Image Encoder）

同时优化三个目标：

目标 说明
ITC（图文对比） 对齐图像和文本的全局表示
ITM（图文匹配） 判断图文是否匹配（二分类）
ITG（图文生成） 根据图像生成对应文字描述

第二阶段：视觉到语言的生成学习（配合冻结的 LLM）

将 Q-Former 的输出通过线性层投影到 LLM 的词嵌入空间，让 LLM 把图像特征当作”软提示”（soft prompt）来生成文本。

3. 与其他方法的对比

方法 可训练参数 视觉编码器 LLM
Flamingo 约 10B 冻结 冻结
LLaVA 全量微调 冻结 微调
BLIP-2 188M 冻结 冻结
InstructBLIP 188M + 指令微调 冻结 冻结

BLIP-2 用不到 2 亿参数的训练量，达到了当时最先进的多模态性能。

5. 局限性

  • Q-Former 的 32 个查询向量可能丢失细节信息（如文字、精确位置）
  • 两阶段训练流程较复杂
  • 对需要精确空间理解的任务（如 OCR、目标检测）效果有限

总结

BLIP-2 的核心贡献是提出了参数高效的多模态对齐范式：冻结两端的大模型，只训练轻量桥接模块。这一思路影响了后续大量工作，包括 InstructBLIP、MiniGPT-4 等。
